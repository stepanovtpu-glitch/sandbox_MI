# GasMeter Pro — инструкция пользователя для пилотной эксплуатации

Документ описывает базовый порядок работы с программой в пилотном режиме.

## 1. Назначение программы

GasMeter Pro предназначен для:

- ведения библиотеки методик измерений;
- хранения скан-копий МИ;
- фиксации SHA-256 скан-копий;
- расчёта метрологических характеристик измерительной линии;
- выбора расчётного шаблона под тип МИ;
- сохранения расчётов в историю;
- выгрузки протоколов PDF/DOCX;
- повторной проверки воспроизводимости расчёта.

## 2. Основной экран

Основной экран состоит из трёх рабочих областей:

1. Левая панель — ввод исходных данных:
   - объект / расчёт;
   - параметры трубопровода;
   - диапазоны Q/P/T;
   - рабочая точка PTZ;
   - выбор МИ;
   - выбор расчётного шаблона.

2. Центральная область:
   - экранная форма УУГ-01;
   - схема измерительной линии;
   - итоговые показатели;
   - структура погрешности / неопределённости;
   - audit log расчёта.

3. Правая панель:
   - состояние системы;
   - итоговый статус расчёта;
   - сохранение расчёта;
   - открытие полного журнала;
   - выгрузка протокола;
   - быстрый список истории;
   - подбор МИ.

## 3. Расчёт новой измерительной линии

1. В поле `Название расчёта` указать объект или краткое описание расчёта.
2. Заполнить параметры трубопровода:
   - Dn трубы;
   - Dn расходомера;
   - прямой участок до расходомера;
   - прямой участок после расходомера.
3. Заполнить диапазоны применимости:
   - Q min / Q max;
   - P min / P max;
   - T min / T max.
4. Заполнить рабочую точку PTZ:
   - рабочий расход;
   - избыточное давление;
   - температуру;
   - атмосферное давление;
   - Z рабочее;
   - Z стандартное.
5. Выбрать методику измерений.
6. Выбрать расчётный шаблон.
7. Проверить результат в центральной области.

## 4. Выбор расчётного шаблона

Доступные шаблоны:

| Шаблон | Назначение |
|---|---|
| DRG_SERIES | Расходомеры ДРГ / PTZ-пересчёт |
| GAS_VOLUME_PTZ | Общий шаблон объёмного расхода газа |
| ROTARY_COUNTER_GAS | Ротационные счётчики газа |
| TURBINE_COUNTER_GAS | Турбинные счётчики газа |
| ULTRASONIC_GAS | Ультразвуковые расходомеры газа |
| MANUAL_QUADRATURE | Ручное квадратурное суммирование |
| CUSTOM | Резерв для пользовательского шаблона |

Для пилота использовать только шаблоны, по которым есть контрольные примеры и ручная сверка.

## 5. Сохранение расчёта

Для сохранения расчёта нажать:

```text
Сохранить расчёт
```

В историю сохраняются:

- дата и время;
- название расчёта;
- выбранная МИ;
- версия МИ;
- выбранный расчётный шаблон;
- исходные данные;
- результат;
- audit log;
- SHA-256 скан-копии МИ, если файл привязан к версии МИ.

## 6. Формирование протокола

На основном экране доступны кнопки:

```text
DOCX
PDF
```

Протокол содержит:

- выбранную МИ;
- расчётный шаблон;
- исходные данные;
- итоговую погрешность / неопределённость;
- предел МИ;
- вклад составляющих;
- audit log;
- заключение.

## 7. Работа с историей

Открыть полный журнал можно через:

```text
История
Журнал
Открыть историю
```

В полном журнале доступны:

- поиск по объекту, МИ, версии, шаблону, заключению;
- фильтр по статусу;
- просмотр карточки расчёта;
- загрузка расчёта обратно в форму;
- выгрузка PDF/DOCX из истории;
- проверка SHA-256 скан-копии МИ.

## 8. Загрузка расчёта из истории

1. Открыть журнал.
2. Выбрать расчёт.
3. Нажать:

```text
Загрузить в форму
```

Программа восстановит:

- параметры линии;
- составляющие погрешности;
- PTZ-контекст;
- МИ;
- расчётный шаблон;
- название расчёта.

После загрузки расчёт автоматически пересчитывается по текущему расчётному модулю.

## 9. Проверка SHA-256 из истории

В карточке сохранённого расчёта нажать:

```text
Проверить SHA-256 МИ
```

Возможные результаты:

| Статус | Значение |
|---|---|
| valid | Текущий файл соответствует сохранённому SHA-256 |
| changed | Файл изменён после сохранения расчёта |
| missing | Файл отсутствует или не найден |

Если SHA-256 изменился, использовать протокол без дополнительной проверки нельзя.

## 10. Работа с библиотекой МИ

Порядок работы:

1. Выбрать МИ.
2. Проверить диапазоны применимости.
3. Загрузить PDF-скан-копию.
4. Проверить SHA-256.
5. Добавить контрольные примеры.
6. При изменении методики создать новую версию.

Запрещается перезаписывать старую версию МИ, если по ней уже были выполнены расчёты.

## 11. Правило пилотного применения

В пилоте использовать только те МИ, по которым выполнено:

- PDF загружен;
- SHA-256 зафиксирован;
- выбран расчётный шаблон;
- внесены контрольные примеры;
- результаты сверены вручную;
- протокол PDF/DOCX успешно формируется.

## 12. Ограничения пилотной версии

Пилотная версия не является утверждённым промышленным средством метрологического расчёта до завершения верификации по реальным МИ.

Результаты программы должны использоваться как расчётно-аналитическая поддержка и подлежат проверке ответственным метрологом.

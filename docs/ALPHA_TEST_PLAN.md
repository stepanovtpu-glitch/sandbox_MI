# План альфа-тестирования GasMeter Pro

Документ фиксирует минимальный объём, который должен быть стабилен перед передачей внутренним пользователям: метрологу, инженеру и администратору пилотной установки.

## Цель альфа-среза

Альфа-версия должна подтвердить, что основной маршрут расчёта УУГ работает без ручной правки данных в БД:

1. инженер задаёт параметры измерительной линии;
2. выбирает СИ из инвентарной базы;
3. выбирает МИ и расчётный шаблон;
4. получает расчёт U / δΣ и скоринг МИ;
5. при превышении допуска получает конкретные замены СИ;
6. применяет замену и пересчитывает результат;
7. сохраняет расчёт в историю;
8. выгружает PDF/DOCX-протокол с составом СИ, версией МИ и SHA-256 документа.

## Что входит в альфу

- Backend FastAPI и frontend React/Vite.
- SQLite-хранилище МИ, версий МИ, истории, audit log и инвентаря СИ.
- Seed-набор СИ: расходомеры, датчики давления, датчики температуры, вычислители и анализатор состава.
- Библиотека МИ серии ДРГ.М с версиями.
- Загрузка PDF МИ и проверка SHA-256.
- Расчётные шаблоны PTZ / DRG_SERIES и ручное квадратурное суммирование.
- Скоринг МИ по Q/P/T и пределу U / δΣ.
- Рекомендации замены СИ из инвентаря при превышении индивидуальных допусков МИ.
- История расчётов и восстановление расчёта из истории.
- PDF/DOCX-протокол расчёта.
- Диагностический экран готовности и отчёт готовности.
- CI: backend tests, frontend build, Docker build.

## Что не входит в альфу

- Electron-инсталлятор.
- Автоматический NLP-парсинг PDF-методик.
- Semantic search / embeddings по базе СИ.
- Локальная LLM и генерация экспертных пояснений.
- Полноценный промышленный workflow утверждения МИ.
- Массовый импорт корпоративного справочника СИ из Excel на сотни записей.

## Обязательные сценарии

| Код | Сценарий | Ожидаемый результат |
|---|---|---|
| A-01 | Открыть приложение и проверить `/health` | Backend отвечает `ok`, frontend загружается |
| A-02 | Выбрать СИ из базы и выполнить расчёт | U / δΣ пересчитывается автоматически |
| A-03 | Искусственно выбрать СИ с превышением допуска | Отображается несоответствие и рекомендация замены |
| A-04 | Применить рекомендуемую замену СИ | Выбранный СИ меняется, расчёт обновляется |
| A-05 | Сохранить расчёт | Запись появляется в истории |
| A-06 | Открыть расчёт из истории | Восстанавливаются параметры линии, МИ, шаблон и СИ |
| A-07 | Сформировать PDF/DOCX | Протокол содержит МИ, SHA-256, состав СИ, вклады и audit log |
| A-08 | Загрузить PDF МИ и проверить SHA-256 | Проверка возвращает `valid` |
| A-09 | Запустить readiness report | Инвентарь СИ и библиотека МИ отражаются в проверках |
| A-10 | Запустить CI-проверки | Backend, frontend и Docker проходят без ошибок |

## Критерии допуска к альфа-тестированию

- `pytest -q` проходит.
- `npm run build` проходит.
- `docker compose -f compose.yaml build` проходит.
- В библиотеке есть минимум 2 реальные МИ для проверки.
- Для основной МИ есть минимум 3 контрольных примера.
- В инвентаре есть минимум 1 доступная замена для давления или температуры.
- PDF/DOCX-протокол включает состав выбранных СИ.
- Известные ограничения описаны в этом документе и не блокируют основной маршрут.

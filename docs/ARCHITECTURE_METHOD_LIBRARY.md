# Архитектура расширяемой библиотеки МИ

## Цель

Новые и обновлённые методики измерений должны добавляться без переделки программы. Код меняется только при появлении принципиально нового расчётного шаблона.

## Разделение ответственности

### Код программы

- хранит универсальные сервисы расчёта;
- хранит расчётные шаблоны;
- выполняет скоринг применимости МИ;
- управляет версиями методик;
- формирует протоколы расчёта;
- хранит аудит и воспроизводимость расчётов.

### Данные МИ

Каждая МИ хранится как версия записи библиотеки:

- регистрационный номер;
- наименование;
- область применения Q/P/T;
- допустимая расширенная неопределённость или погрешность;
- требования к СИ;
- тип расходомера;
- расчётный шаблон;
- PDF-документ;
- контрольные примеры;
- статус версии.

## Расчётные шаблоны

Первый реализуемый шаблон:

- `DRG_SERIES` — серия ДРГ.М-160 / 800 / 1600 / 2500.

Базовые универсальные шаблоны:

- `MANUAL_QUADRATURE` — универсальное квадратурное суммирование;
- `CUSTOM` — резерв для новых типов методик.

## Версионирование

МИ нельзя перезаписывать. Новая редакция создаёт новую версию:

```text
0168/RA.RU.314369/2024:v1 — архив
0168/RA.RU.314369/2026:v2 — активная
```

Каждый расчёт должен хранить ссылку на конкретную версию МИ, чтобы результат можно было воспроизвести через несколько лет.

## API

```text
GET /api/methods
GET /api/methods/{mi_id}/versions
POST /api/methods/{mi_id}/versions
POST /api/methods/score
POST /api/calculate
```

## Временная реализация

На текущем этапе библиотека МИ хранится in-memory и загружается из seed-данных. Следующий этап — перенос в SQLite с миграциями и файловым хранилищем документов МИ.

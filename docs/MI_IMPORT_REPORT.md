## Что сделано

- PDF скопированы в хранилище приложения `backend/data/method_documents`.
- Для каждой активной версии рассчитан и сохранён SHA-256.
- Регистрационные номера приведены к виду `NNNN.RA.RU.314369/2024`.
- Для ДРГ.М и файлов с явным `до N м3` предварительно заполнен диапазон расхода из имени файла.
- Для файлов без однозначного расходного диапазона установлен безопасный технический диапазон `0.0-0.001 м3/ч`, чтобы методика была в библиотеке, но не подбиралась как подходящая до OCR-сверки.

## Ограничение

Все проверенные PDF являются скан-копиями без текстового слоя: `pdfplumber` извлекает 0 символов с первых страниц. Поэтому извлечение реквизитов и диапазонов выполняется через встроенный OCR-контур Tesseract, а не через обычное чтение PDF-текста.

## OCR-обработка

Обновлено: 2026-06-20

- Установлен внешний OCR-движок Tesseract OCR 5.4.0.
- Русская и английская языковые модели включены в состав backend: `backend/app/ocr_tessdata`.
- В backend добавлен штатный OCR-сервис и endpoint `POST /api/methods/{mi_id}/versions/{version_id}/document/ocr`.
- Добавлена CLI-команда пакетной обработки: `backend\ocr_methods.py`.
- OCR первого листа выполнен для 36 активных МИ.
- Результат: `recognized=26`, `needs_review=10`, `poor=0`.
- Для распознанных МИ автоматически заполнен диапазон Q из скана свидетельства.
- Для МИ со статусом `needs_review` текст распознан, но ключевой диапазон не извлечён автоматически; требуется ручная сверка или OCR дополнительных страниц.

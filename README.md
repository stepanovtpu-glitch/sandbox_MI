# GasMeter Pro

GasMeter Pro — прототип программы для подбора методик измерений, расчёта погрешности / расширенной неопределённости, ведения библиотеки МИ, хранения скан-копий и формирования протоколов расчёта.

## Что уже реализовано

- Backend на FastAPI.
- Frontend на React + Vite.
- Библиотека методик измерений с версиями.
- Хранение данных в SQLite.
- Загрузка PDF-скан-копий МИ.
- Контроль SHA-256 для скан-копий МИ.
- Проверка целостности PDF-файла по SHA-256.
- Расчётный шаблон DRG_SERIES.
- PTZ-пересчёт и аудит расчёта.
- Контрольные примеры МИ.
- История расчётов.
- Экспорт протокола расчёта в PDF и DOCX.
- Диагностический блок состояния системы.
- Инвентарная база средств измерений для альфа-сценариев.
- Рекомендации замены СИ из инвентаря при превышении индивидуальных допусков МИ.
- CI-проверка backend, frontend и Docker-сборки.

## Быстрый запуск через Docker

Требуется Docker Desktop или Docker Engine с поддержкой Docker Compose.

```bash
docker compose -f compose.yaml up --build
```

После запуска открыть:

```text
http://localhost:5173
```

Backend API будет доступен по адресу:

```text
http://localhost:8000
```

Проверка backend:

```text
http://localhost:8000/health
```

Диагностика backend и базы:

```text
http://localhost:8000/api/system/info
```

## Локальное хранение данных

Рабочие данные сохраняются в папке:

```text
backend/data/
```

Структура:

```text
backend/data/
  gasmeter.db
  method_documents/
    <mi_id>/
      <version_id>/
        <scan>.pdf
  reports/
    calculation_protocol_*.pdf
    calculation_protocol_*.docx
```

Назначение:

- `gasmeter.db` — база SQLite с МИ, версиями, историей расчётов и служебными данными.
- `method_documents/` — скан-копии МИ, привязанные к версиям.
- `reports/` — сформированные протоколы расчёта.

Эта папка не должна храниться в GitHub, так как содержит рабочие файлы, сканы и расчёты.

## Запуск без Docker

### Backend

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

Для Linux / macOS активация окружения:

```bash
source .venv/bin/activate
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

Открыть:

```text
http://localhost:5173
```

## CI GitHub Actions

В репозитории настроен workflow:

```text
.github/workflows/ci.yml
```

Он запускает:

- backend tests;
- frontend build;
- docker build.

Если один из блоков падает, изменения нельзя считать готовыми к передаче.

## Работа с МИ

1. Открыть блок `Библиотека МИ`.
2. Выбрать методику.
3. Проверить карточку МИ: регистрационный номер, диапазоны Q/P/T, предел U/δΣ.
4. Загрузить PDF-скан-копию к активной версии.
5. При необходимости нажать `Проверить файл МИ по SHA-256`.
6. Добавить контрольные примеры.
7. При изменении требований создать новую версию МИ.

Важно: старая версия МИ не должна перезаписываться. Для воспроизводимости расчётов создаётся новая версия.

## История расчётов

Расчёт можно сохранить в историю. В записи сохраняются:

- название проекта / объекта;
- дата расчёта;
- выбранная МИ;
- версия МИ;
- SHA-256 скан-копии;
- итоговая U/δΣ;
- предел МИ;
- статус;
- полный запрос и результат расчёта.

## Протокол расчёта

Доступны форматы:

- PDF;
- DOCX.

В протокол включаются:

- выбранная МИ;
- регистрационный номер;
- версия МИ;
- сведения о скан-копии;
- SHA-256;
- диапазоны Q/P/T;
- расчётный шаблон;
- итоговая U/δΣ;
- вклад составляющих;
- аудит расчёта;
- заключение.

## Текущий статус

Это рабочий MVP-каркас. Для промышленного применения требуется дальнейшая верификация расчётных формул по каждой реальной МИ и расширение набора контрольных примеров.
